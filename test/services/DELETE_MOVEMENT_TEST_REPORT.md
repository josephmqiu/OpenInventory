- 库存应恢复为 50
- 记录应从数据库中移除

## 二、多记录场景测试

### 2.1 多物料独立删除测试
**测试目标**: 验证删除一个物料的记录不影响其他物料

**输入条件**:
- 物料 A: 库存 100, 入库记录 +50
- 物料 B: 库存 200, 入库记录 +100
- 物料 C: 库存 150, 入库记录 +75

**执行步骤**:
1. 创建三个物料及其入库记录
2. 仅删除物料 B 的入库记录

**预期结果**:
- 物料 A 库存: 100 (不变)
- 物料 B 库存: 100 (恢复)
- 物料 C 库存: 150 (不变)
- 物料 A 和 C 的记录应仍然存在

**验证标准**: ✅ 通过

### 2.2 后续记录重新计算测试
**测试目标**: 验证删除记录后，后续记录的库存数量正确更新

**输入条件**:
- 记录 1: 入库 +50 (50 → 100) [10:00]
- 记录 2: 入库 +30 (100 → 130) [11:00] ← 删除此记录
- 记录 3: 入库 +20 (130 → 150) [12:00]

**执行步骤**:
1. 创建三条顺序入库记录
2. 删除第二条记录

**预期结果**:
- 库存总数: 120 (150 - 30)
- 记录 3 的 previous_quantity: 100 (原 130)
- 记录 3 的 new_quantity: 120 (原 150)

**验证标准**: ✅ 通过

### 2.3 数据完整性保护测试
**测试目标**: 验证删除操作不影响未删除记录的数据完整性

**输入条件**:
- 记录 1: 入库 +50 (50 → 100)
- 记录 2: 入库 +50 (100 → 150) ← 删除此记录
- 记录 3: 入库 +50 (150 → 200)

**执行步骤**:
1. 创建三条记录
2. 删除中间记录
3. 验证记录 1 和记录 3 的数据完整性

**预期结果**:
- 记录 1 的 previous_quantity: 50 (不变)

## 发现的问题与修复

### 问题 1: 后续记录 previous_quantity 计算错误
**描述**: 删除记录后，后续记录的 previous_quantity 未正确设置为被删除记录的 previous_quantity

**影响**: 导致历史数据中的库存数量不正确

**修复方案**: 
- 使用被删除记录的 `previous_quantity` 作为新的库存基准
- 修改 `newQuantity = movement.previous_quantity`

**状态**: ✅ 已修复

### 问题 2: 后续出库记录可能导致负库存
**描述**: 删除入库记录后，未检查后续出库记录是否会导致负库存

**影响**: 可能出现库存数量为负数的非法状态

**修复方案**:
- 在更新后续记录时添加库存检查
- 如果出库数量大于当前库存，抛出错误

**状态**: ✅ 已修复

### 问题 3: 操作顺序问题
**描述**: 查询后续记录的操作在删除记录之后执行

**影响**: 可能无法正确获取所有需要更新的记录

**修复方案**:
- 将查询后续记录的操作移到删除记录之前

**状态**: ✅ 已修复
